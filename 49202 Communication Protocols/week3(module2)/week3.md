### ① 상단 패널 — Packet List (패킷 목록)
캡처된 모든 패킷을 한 줄씩 요약해서 보여주는 곳

- **No.** — 캡처된 패킷 순서 번호. 1번부터 시작해서 올라감
- **Time** — 캡처 시작 시점을 0초로 기준잡고, 그로부터 경과한 시간(초). 268.672077024 이면 캡처 시작 후 268.67초 뒤에 이 패킷이 잡혔다는 의미
- **Source** — 패킷을 보낸 쪽 주소. Layer 2 패킷(ARP 등)이면 MAC 주소(86:57:f9:68:9f:98 처럼 콜론으로 구분된 6쌍 16진수)로 표시되고, Layer 3 패킷이면 IP 주소로 표시됨
- **Destination** — 패킷을 받는 쪽 주소. Broadcast는 특정 장치가 아니라 같은 네트워크 안의 모든 장치에게 동시에 뿌리는 것. 내가 누군지 모르니까 전체한테 물어보는 방식
- **Protocol** — 이 패킷이 사용하는 프로토콜. ARP는 IP 주소는 아는데 MAC 주소를 모를 때 전체에 브로드캐스트로 물어보는 프로토콜. "10.0.3.254 IP 가진 장치야, MAC 주소 뭐야?" 라고 묻는 것. 이외에 DNS, TCP, UDP, HTTP 등이 있음
- **Length** — 패킷 크기. 단위는 바이트(byte). 42이면 42바이트짜리 패킷이라는 의미
- **Info** — 패킷 내용 한줄 요약. "Who has 10.0.3.254? Tell 10.0.3.1"은 "10.0.3.254 IP를 가진 장치야, 네 MAC 주소를 10.0.3.1한테 알려줘"라는 뜻

### ② 하단 좌측 패널 — Packet Details (패킷 상세)
선택한 패킷 하나를 OSI 레이어별로 쪼개서 보여주는 곳. 각 항목 왼쪽 ▶ 클릭하면 세부 필드가 펼쳐짐

- **Frame (Layer 1 - Physical)** — 패킷이 물리적으로 전송될 때의 정보. 전체 크기, 캡처된 크기, 캡처 시각 등
- **Ethernet II (Layer 2 - Data Link)** — MAC 주소 기반 통신 정보. 출발지/목적지 MAC 주소 포함. MAC 주소는 네트워크 장치 하나하나에 부여된 고유 물리 주소
- **ARP / IP (Layer 3 - Network)** — ARP면 IP↔MAC 주소 매핑 정보. IP 패킷이면 출발지/목적지 IP 주소, TTL(패킷이 몇 개의 라우터를 거칠 수 있는지 제한하는 숫자) 등
- **TCP / UDP (Layer 4 - Transport)** — 포트 번호(어떤 프로그램한테 전달할지), TCP면 SYN/ACK 같은 연결 상태 정보. 현재 화면엔 ARP만 있어서 이 레이어는 안 보임
- **HTTP / DNS (Layer 7 - Application)** — 실제 웹 요청 내용, DNS 쿼리(도메인→IP 변환 요청) 등. Firefox로 사이트 접속하면 이 레이어가 보임

### ③ 하단 우측 패널 — Packet Bytes (원시 데이터)
패킷의 실제 날것 데이터를 16진수와 ASCII로 보여주는 곳. 레이어 구분 없이 패킷 전체를 바이트 단위로 표시

- **왼쪽 숫자열 (0000, 0010, 0020...)** — 오프셋(offset). 패킷 데이터의 시작점(0)으로부터 몇 번째 바이트인지를 나타내는 위치값. 0000은 첫 번째 줄, 0010은 16번째 바이트부터 시작하는 줄이라는 의미. 16진수로 표기됨
- **가운데 16진수 (ff ff ff ff 86 57...)** — 실제 패킷 데이터를 16진수로 표현한 것. 2자리가 1바이트. ff ff ff ff ff ff는 Broadcast MAC 주소를 의미함. 좌측 패널에서 항목 클릭하면 해당 바이트가 파란색으로 하이라이트됨
- **오른쪽 ASCII (......Wh...)** — 가운데 16진수 데이터를 사람이 읽을 수 있는 문자로 변환해서 보여주는 것. 변환이 안 되는 바이트는 점(.)으로 표시. HTTP처럼 텍스트 기반 프로토콜은 여기서 실제 내용이 보임
